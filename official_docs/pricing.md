# Token consumption-based billing
<span id="d1a2d969"></span>
## Token usage explanation
A key concept in model service usage is the token. When you call a model inference API, the input content is tokenized—converted into units the model can understand. After processing, the model outputs tokens that are then converted back into text or other content formats. The number of tokens the model processes—both input and output—is a primary billing unit for model inference services.
Below is the typical fee calculation method for the model inference service.
```Plain Text
Cost = Number of tokens used × Price per token
```


* Due to the different tokenizers used in different models, the same piece of text may correspond to different numbers of tokens.
* You can use the [Tokenizer tool](https://console.byteplus.com/ark/region:ark+ap-southeast-1/tokenCalculator) or <a href="https://docs.byteplus.com/en/docs/ModelArk/tokenization">Tokenization API</a> to estimate the token consumption of content.

## LLMs & VLMs
<span id="c0c15831"></span>
### Online inference

* Pay-as-you-go, with fees charged hourly based on input and output tokens.
* Billing by token applies to seed and skylark LLMs.
* If **context caching** is enabled, cost is calculated using the following formula:

```Plain Text
Total cost of an online inference = 
    Input unit price × (Input token - Cache-hit token) + 
    Cache-hit unit price × Cache-hit token + 
    Cache storage unit price × Cache storage token × Duration + 
    Output unit price × Output token
```


   See [Context caching](/docs/ModelArk/1330310#476e6f25) for models that support this feature.
* Some models support tiered unit pricing based on the input length (and output length) of each request, as shown in the table below.
   For example, when calling the seed-1-6 model, if a request has an input length of 200k tokens and an output length of 14k tokens, it falls within the input length tier (128k, 256k]. In this case, all tokens generated by the model are billed at the following unit prices:
   *  Input: USD 0.50 per million tokens
   *  Cache-hit input: USD 0.05 per million tokens
   *  Output: USD 4.00 per million tokens

* Pay-as-you-go; for some models, batch inference is priced at only 50% of the online inference price.
* Cost is calculated using the following formula:

```Plain Text
Batch inference fee = 
    Input unit price × (Input tokens excluding cache-hit tokens) + 
    Cache-hit unit price × Cache-hit tokens + 
    Output unit price × Output tokens
```

## Video generation
> Asynchronous inference: Billing is based on the token equivalent of the output video.


* **Inference Cost = Token Unit Price × Video Token Consumption**
* The video token consumption is approximately calculated as:
   * **Standard Video**：Token Consumption ≈ (Width × Height × Frame Rate × Duration) / 1024
   * **Draft Video (only 480p)**：Token Consumption ≈  (Width × Height × Frame Rate × Duration) / 1024 × Conversion coefficient；The conversion coefficients for **Seedance 1.5 Pro** are as follows: 0.7 for video without audio; 0.6 for video with audio.

:::warning
* Charges apply only to successfully generated videos. No fees are incurred for videos that fail to generate due to reasons such as moderation.
* The actual number of tokens consumed is determined by the `usage` field in the response returned by the API call.
:::

# Rate limits
The call limits for each model are requests per minute (RPM) and tokens per minute (TPM). You can go to the [Model activation](https://console.byteplus.com/ark/region:ark+ap-southeast-1/openManagement) page to view the limits of each account (along with all its sub-accounts).
To increase your limits, submit a request on the console.
<span id="a05784b7"></span>
# Service unavailability

* An account is considered in arrears when its balance becomes insufficient or falls below zero.
* If an account stays in arrears for 2 hours, the model services will be suspended. Services will resume once the account has been recharged. Please make sure that all outstanding bills are paid on time to avoid any unexpected service interruption.
